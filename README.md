# 🧠 Backend Handcoding Practice (Nest + Spring)

이 문서는 손코딩 기반으로 백엔드 핵심 개념을 단계적으로 학습하기 위한 커리큘럼입니다.

목표:
- 프레임워크 문법이 아니라 **백엔드 공통 구조 이해**
- AI 없이 직접 구현 → 검수 → 회고
- NestJS와 Spring Boot로 동일 기능을 각각 구현

# 1️⃣ 회원 생성 (Validation + 중복 체크)

## 요구사항
- 이름 필수
- 이메일 필수
- 이메일 형식 검증
- 이메일 중복 불가

## 핵심 개념
- 입력 검증
- 중복 데이터 방지

---

# 2️⃣ 회원 단건 조회 (조회 + 예외 처리)

## 요구사항
- id로 회원 조회
- 존재하지 않으면 404
- 민감 정보 제외

## 핵심 개념
- 조회 로직
- 예외 처리
- 응답 DTO 분리

---

# 3️⃣ 작업 생성 (비즈니스 로직)

## 요구사항
- 제목 필수
- 설명 선택
- 상태 기본값 TODO
- 마감일은 과거 불가
- 담당자(userId) 존재해야 함

## 핵심 개념
- 비즈니스 규칙
- 연관 데이터 검증

---

# 4️⃣ 작업 상태 변경 (상태 머신)

## 요구사항
- 상태: TODO / IN_PROGRESS / DONE
- DONE → TODO 변경 금지

## 핵심 개념
- 상태 전이
- enum 활용

---

# 5️⃣ 담당자별 작업 조회 (조건 조회)

## 요구사항
- assigneeId로 필터
- 상태별 필터
- 최신순 정렬

## 핵심 개념
- query parameter
- 조건 조회

---

# 6️⃣ 페이징 조회 (성능)

## 요구사항
- page, size 입력
- 기본값 설정
- totalCount 포함
- 정렬 기준 설정

## 핵심 개념
- pagination
- limit / offset

---

# 7️⃣ 댓글 생성 (연관관계)

## 요구사항
- 작업(Task) 존재해야 함
- 사용자(User) 존재해야 함
- 내용 필수

## 핵심 개념
- 다대일 관계
- 부모 엔티티 검증

---

# 8️⃣ 소프트 삭제 (데이터 전략)

## 요구사항
- deletedAt 필드 사용
- 삭제된 데이터 조회 제외
- 중복 삭제 방지

## 핵심 개념
- soft delete
- 데이터 보존 전략

---

# 9️⃣ 권한 체크 (인가)

## 요구사항
- 작성자만 수정 가능
- 다른 사용자는 수정 불가

## 핵심 개념
- authorization
- 접근 제어

---

# 🔟 트랜잭션 (데이터 일관성)

## 요구사항
- 작업 생성 + 로그 생성
- 하나라도 실패 시 전체 롤백

## 핵심 개념
- transaction
- rollback

---

# 🔥 학습 방법 (중요)

## 1. 설계 먼저
코드 치기 전에 반드시 작성:

- 입력값
- 검증 조건
- 처리 로직
- 출력값

---

## 2. 손코딩
- 최대한 참고 없이 구현
- 막히면 그때만 문서 확인

---

## 3. 테스트
- 정상 케이스
- 실패 케이스
- 경계값 테스트

---

## 4. AI 리뷰 (마지막 단계만)

예시 질문:
- validation 빠진 부분 있나?
- transaction 필요한가?
- 실무에서 문제될 구조인가?

---

## 5. 회고

기록할 것:
- 막힌 부분
- 이해 안 된 개념
- 다음에 개선할 점
